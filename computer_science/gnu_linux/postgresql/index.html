    PostgreSQL

    Usage‚öë

    Use psql.

    Queries ignore case, so if any column name has capital letters in it, escape it with double quotes ("). To scape a string use single quotes (').

    To select the database to work in do

    \connect {database_name}

    or \c {database_name} in short.

    Basic operations‚öë

    Delete a table‚öë

    DROP TABLE "{table_name}";

    Add a new column‚öë

    To add a new column, use ALTER TABLE.

    For example:

    ALTER TABLE {table_name}
  ADD IF NOT EXISTS {column_name} {data_type}

    GROUP BY‚öë

    Reference: PostgreSQL GROUP BY

    UPDATE‚öë

    To update the values of some columns, use UPDATE.

    The general basic syntax is as follows:

    UPDATE table_name
SET column1 = value1,
    column2 = value2,
    ...
WHERE condition;
    From subquery‚öë

    The most efficient way appears to be the following.

    UPDATE {table}
SET {column1}=subquery.{column1},
    {column2}=subquery.{column2}
FROM ({subquery}) AS subquery
WHERE {table}.{id} = subquery.{id};

    Other options are to perform a JOIN but the syntax is less clear and the performance seems to be worse.

    Window functions‚öë

    Window functions allow to perform the calculation across a set of rows related to the current row.

    The simplified syntax is:

    window_function(arg1, arg2,..) OVER (
   [PARTITION BY partition_expression]
   [ORDER BY sort_expression [ASC | DESC] [NULLS {FIRST | LAST }])

    So for example a moving average of the last 28 days could be obtained with:

    SELECT
    AVG({column})
        OVER (
         ORDER BY {date_column}
               RANGE BETWEEN '28 day' PRECEDING AND current row)
         AS rolling_average
 FROM
     {table}

    Query operators‚öë

    NULLIF‚öë

    The NULLIF(value1, value2) function returns a null value if value1 equals value2; otherwise it returns value1.

    It can be useful for avoiding divisions by 0 (e.g. set value2 to 0).

    String functions and operators‚öë

    Reference: PostgreSQL Documentation

    concat_ws‚öë

    To concatenate strings with a separator, use concat_ws({separator}, {val1}, {...}).

    For example: concat_ws(',', 'abcde', 2, NULL, 22) ‚Üí abcde,2,22.

    Time stamp operations‚öë

    Get only part of the time stamp‚öë

    To get only a part of the time stamp (e.g., day, hour...) or to get it in another format (e.g., epoch), use date_part(). For example, to get the hour of a timestamp do:

    date_part('hour', timestamp '2001-02-16 20:38:40')

    which will return 20. The second argument of the function can be the name of a column.

    If instead you want to truncate a timestamp to a specified level of precission, use date_trunc('datepart', field).

    Meta‚öë

    Get column names of a table‚öë

    SELECT column_name FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME = '{table_name}';
    Or, to get the column names along with their type use:

    \d+ {table_name}

    Enable command timing‚öë

    \timing [on|off]
    • stackoverflow

    Stop/kill process‚öë

    First locate the process pid with

    SELECT pid, query FROM pg_stat_activity WHERE state = 'active';

    Then, stop or kill the process with SELECT pg_cancel_backend({pid}) or pg_termiante_backend({pid}) respectively.

    Configuration‚öë

    You can use PGTune to calculate configuration for PostgreSQL based on the maximum performance for a given hardware configuration.

    Parallelization‚öë

    To tune the number of workers edit postgresql.conf and in the - Asynchronous Behavior - section edit max_worker_processes, max_parallel_workers_per_gather and max_parallel_workers.

    Administration‚öë

    Change user password‚öë

    ALTER USER user_name WITH PASSWORD 'new_password';
